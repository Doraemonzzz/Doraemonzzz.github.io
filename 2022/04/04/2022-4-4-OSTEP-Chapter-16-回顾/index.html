第16 章 分段

内容回顾

  • 分段：

    • 在MMU中引入不止一个基址和界限寄存器，给地址空间内的每个逻辑段（segment）一对

  • 端：

    • 地址空间里一个连续定长的区域
    • 典型的地址空间里有3个逻辑不同的段：代码，栈，堆

  • 段错误（segmentation fault）：

    • 在支持分段的机器上发生了非法的内存访问

  • 判断引用了哪个段

    • 显式方法

      • 把虚拟地址的前几位来表示段，后几位来表示偏移量

        // get top 2 bits of 14-bit VA
Segment = (VirtualAddress & SEG_MASK) >> SEG_SHIFT
// now get offset
Offset = VirtualAddress & OFFSET_MASK
if (Offset >= Bounds[Segment])
	RaiseException(PROTECTION_FAULT)
else
	PhysAddr = Base[Segment] + Offset
	Register = AccessMemory(PhysAddr)

    • 隐式

      • 通过地址产生的方式来确定段

    • 共享

      • 地址空间之间共享（share）某些内存段

    • 粗粒度和细粒度的分段

    • 操作系统对分段的支持

      • 上下文切时需要保存和恢复各个段寄存器中的内容
      • 管理物理内存的空闲空间
        • 会产生外部碎片的问题

问题1：

λ python ./segmentation.py -a 128 -p 512 -b 0 -l 20 -B 512 -L 20 -s 0
ARG seed 0
ARG address space size 128
ARG phys mem size 512

Segment register information:

  Segment 0 base  (grows positive) : 0x00000000 (decimal 0)
  Segment 0 limit                  : 20

  Segment 1 base  (grows negative) : 0x00000200 (decimal 512)
  Segment 1 limit                  : 20

Virtual Address Trace
  VA  0: 0x0000006c (decimal:  108) --> PA or segmentation violation?
  VA  1: 0x00000061 (decimal:   97) --> PA or segmentation violation?
  VA  2: 0x00000035 (decimal:   53) --> PA or segmentation violation?
  VA  3: 0x00000021 (decimal:   33) --> PA or segmentation violation?
  VA  4: 0x00000041 (decimal:   65) --> PA or segmentation violation?

For each virtual address, either write down the physical address it translates to
OR write down that it is an out-of-bounds address (a segmentation violation). For
this problem, you should assume a simple address space with two segments: the top
bit of the virtual address can thus be used to check whether the virtual address
is in segment 0 (topbit=0) or segment 1 (topbit=1). Note that the base/limit pairs
given to you grow in different directions, depending on the segment, i.e., segment 0
grows in the positive direction, whereas segment 1 in the negative.

计算：

VA  0: 0x0000006c (decimal:  108) --> seg 1, offset 20, 512 - 20 = 492
VA  1: 0x00000061 (decimal:   97) --> seg 1, offset 33, segmentation violation
VA  2: 0x00000035 (decimal:   53) --> seg 0, offset 53, segmentation violation
VA  3: 0x00000021 (decimal:   33) --> seg 0, offset 33, segmentation violation
VA  4: 0x00000041 (decimal:   65) --> seg 1, offset 63, segmentation violation

答案：

λ python ./segmentation.py -a 128 -p 512 -b 0 -l 20 -B 512 -L 20 -s 0 -c

ARG phys mem size 512

  Segment 1 base  (grows negative) : 0x00000200 (decimal 512)
  Segment 1 limit                  : 20

Virtual Address Trace
  VA  0: 0x0000006c (decimal:  108) --> VALID in SEG1: 0x000001ec (decimal:  492)
  VA  1: 0x00000061 (decimal:   97) --> SEGMENTATION VIOLATION (SEG1)
  VA  2: 0x00000035 (decimal:   53) --> SEGMENTATION VIOLATION (SEG0)

3

翻译有问题，原文是：

valid, valid, violation, ..., violation, valid, valid

参考资料：

  • https://pages.cs.wisc.edu/~remzi/OSTEP/vm-segmentation.pdf

除了0, 1, 14, 15，都是valid：

b0 0
l0 2

b1 16
l1 2

验证：
